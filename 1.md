# 前端增删改按钮 API 改造方案

## Context

当前项目前端已部分接入后端 API（`lib/api/`），但仍有 6 类操作使用本地模拟（`console.log` + `alert` 或 `setTimeout`），需要逐一改造为调用真实 API。

## 现状总结

| 实体 | 增 | 删 | 改 | 特殊操作 |
|------|:--:|:--:|:--:|----------|
| 需求 | ✅ | ✅ 列表页 | ❌ 本地模拟 | 转IR/拆解/转任务均本地模拟 |
| 任务 | ✅ | ✅ | ✅ | ✅ |
| 项目 | ✅ | ✅ | ✅ | ❌ 成员增删本地操作 |
| 版本 | ✅ | ✅ | ✅ | ✅ |
| 迭代 | ✅ | ✅ | ✅ | ✅ |
| 认证 | ✅ | - | ❌ 改密本地模拟 | - |

## 待改造清单（共 6 项）

### 1. 需求编辑（2 处）

**文件 1：`components/requirements-table.tsx` 第 111-117 行**

当前代码：
```typescript
const handleEditSave = (data: Partial<RequirementVO>) => {
  console.log("保存编辑:", data)
  if (onUpdate && data.id) {
    onUpdate(data as Requirement)
  }
  alert("需求已更新")
}
```

改造为：
```typescript
const handleEditSave = async (data: Partial<RequirementVO>) => {
  try {
    await updateRequirement(data.id as number, data as RequirementUpdateRequest)
    if (onUpdate) onUpdate(data as RequirementVO)
  } catch (err) {
    alert(err instanceof Error ? err.message : "更新失败")
  }
}
```
- 新增导入：`import { updateRequirement } from "@/lib/api/requirements"` + `import type { RequirementUpdateRequest } from "@/lib/api/requirements"`

**文件 2：`app/requirements/[id]/page.tsx` 第 834-837 行**

当前代码：
```typescript
onSave={(data) => {
  console.log("保存需求:", data)
  setEditDialogOpen(false)
}}
```

改造为：
```typescript
onSave={async (data) => {
  try {
    await updateRequirement(requirement.id, data as RequirementUpdateRequest)
    setEditDialogOpen(false)
    // 重新加载数据
    const [req, anc] = await Promise.all([
      getRequirement(requirement.id),
      getAncestors(requirement.id),
    ])
    setRequirement(req)
    setAncestors(anc)
  } catch (err) {
    alert(err instanceof Error ? err.message : "更新失败")
  }
}}
```
- 新增导入：`import { updateRequirement } from "@/lib/api/requirements"` + `import type { RequirementUpdateRequest } from "@/lib/api/requirements"`

### 6. 修改密码（1 处）

**文件：`components/admin-layout.tsx` 第 114-122 行（ChangePasswordDialog 内）**

当前代码：
```typescript
const handleSubmit = async () => {
  if (!oldPassword || !newPassword || !confirmPassword) { alert("请填写完整信息"); return }
  if (newPassword !== confirmPassword) { alert("两次输入的新密码不一致"); return }
  if (newPassword.length < 6) { alert("新密码长度不能少于6位"); return }
  setSaving(true)
  await new Promise(resolve => setTimeout(resolve, 500))
  setSaving(false); setOldPassword(""); setNewPassword(""); setConfirmPassword(""); onOpenChange(false)
  alert("密码修改成功！")
}
```

前置步骤：需先在 `lib/api/auth.ts` 中新增 `changePassword` 函数：
```typescript
export async function changePassword(oldPassword: string, newPassword: string) {
  return client.post<void>('/auth/change-password', { oldPassword, newPassword })
}
```

改造为：
```typescript
const handleSubmit = async () => {
  if (!oldPassword || !newPassword || !confirmPassword) { alert("请填写完整信息"); return }
  if (newPassword !== confirmPassword) { alert("两次输入的新密码不一致"); return }
  if (newPassword.length < 6) { alert("新密码长度不能少于6位"); return }
  setSaving(true)
  try {
    await changePassword(oldPassword, newPassword)
    setSaving(false); setOldPassword(""); setNewPassword(""); setConfirmPassword(""); onOpenChange(false)
    alert("密码修改成功！")
  } catch (err) {
    setSaving(false)
    alert(err instanceof Error ? err.message : "密码修改失败")
  }
}
```
- 新增导入：`import { changePassword } from "@/lib/api/auth"`

## 验证方式

1. 启动后端（`pnpm dev` 或对应后端启动脚本）
2. 逐项测试每个改造后的按钮：
   - **需求编辑**：列表页和详情页分别编辑需求，确认调用 `PUT /requirements/{id}` 且页面数据刷新
   - **LMT转IR**：点击"转换为IR"，确认调用 `POST /requirements/{id}/convert-to-ir`，弹窗关闭
   - **需求拆解**：IR/SR 点击"拆解为SR/AR"，确认调用 `POST /requirements/{id}/decompose`
   - **LMT转任务**：详情页"转为任务"，确认调用 `POST /tasks`
   - **项目成员**：新增/删除成员，确认调用 `POST/DELETE /projects/{id}/members`
   - **修改密码**：确认调用 `POST /auth/change-password`
   - **详情页删除**：确认调用 `DELETE /requirements/{id}` 并跳转回列表
3. 验证错误处理：后端不可达时，应显示错误提示（alert），不静默失败
